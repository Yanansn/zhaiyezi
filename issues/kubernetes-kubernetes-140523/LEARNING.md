# Learning

## Concepts

- Issue ownership is live coordination state, not static metadata. It must be rechecked immediately before investing in implementation.
- `superseded` records that the local contribution stopped because another contributor took ownership; it does not change or close the upstream Issue.

## Reusable methods

- Cross-check assignees, comments, labels, linked pull requests and CI against GitHub whenever resuming an issue.
- Treat a new assignee as a material screening change and prefer a clean handoff over duplicate implementation.
- Record terminal outcomes even when no code was written so future sessions do not accidentally restart abandoned work.
