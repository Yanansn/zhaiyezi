# DECISION-0001：项目运行模型

## 决定

《摘叶子》采用“通用 Skill + 主 Agent + 项目事实库 + 实时 GitHub 状态”的工作方式。

核心 Skill 保持开源项目无关。项目或生态特有规则只有在真实贡献中被验证为稳定、可复用后，才增加为按需加载的 profile。

## 原因

- Agent 负责调查、推理、实现和推进。
- Skill 负责约束流程、质量门槛和记录规范。
- Git 仓库保存可查询、可审计、可继续的长期事实。
- GitHub 是 Issue、PR、CI 和维护者意见的实时事实来源。

## 外部操作边界

分析、代码调查、本地修改、测试和记录可以自动进行。公开评论、认领 Issue、推送分支、创建 PR 和回复维护者必须先取得用户确认。
